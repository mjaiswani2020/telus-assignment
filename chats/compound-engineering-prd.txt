  ---
  Research Synthesis

  Key Findings from Papers

  Anthropic (2204.05862v1) — "Training a Helpful and Harmless Assistant with
  RLHF":
  - Core format: pairwise comparison in multi-turn, real-time conversations with
   models
  - Two task modes: helpfulness (pick better response) and red-teaming (pick
  more harmful response)
  - Preference strength captured but only used to filter near-ties
  - Lightweight QC: writing-quality spot-checks, no IAA-based filtering. ~63%
  agreement was "sufficient"
  - Weekly model hot-swapping during online data collection phase
  - Annotators could edit model responses (late addition, <10% of data)
  - 3 data tranches: base (86k), rejection sampling (54k), online (22k)

  Meta/Llama 2 (2307.09288v2) — "Llama 2: Open Foundation and Fine-Tuned Chat
  Models":
  - Also pairwise comparison, but with a 4-point preference degree scale
  (significantly better → negligibly better)
  - Preference margin fed directly into reward model loss — richer signal
  - Two separate reward models (helpfulness RM + safety RM)
  - Rigorous 4-step annotator qualification process + manual QA review by
  content managers
  - Annotators write their own prompts (not selected from a pool)
  - 14 weekly batches, 1.4M+ comparisons total
  - Safety annotations include a 3-bin safety label + risk category taxonomy
  - Curriculum strategy: simple prompts early, complex later
  - Key insight: "Quality is all you need" — 27k high-quality SFT annotations
  beat millions of low-quality ones

  Proposed Goals & Use Cases

  Based on the research, here are the core goals and use cases I see for RLHF
  DataForge:

  Goals

  1. Methodology-agnostic: Support Anthropic-style (real-time multi-turn),
  Meta-style (batch pre-generated), and hybrid approaches from a single
  configurable platform
  2. Full RLHF data lifecycle: Prompt curation → response generation →
  annotation → quality analysis → export → iteration
  3. Configurable annotation dimensions: Binary preference, n-point scales,
  multi-dimensional rubrics, safety labels, free-text justifications — all
  mix-and-matchable per task
  4. Production-grade quality pipeline: From lightweight crowd approaches to
  rigorous qualification-gated expert workflows
  5. Training-framework-native export: DPO, PPO, reward model formats out of the
   box (TRL/HuggingFace compatible)

  Core Use Cases (derived from papers)

  1. SFT data collection — Annotators write prompt-response pairs (Meta Stage 1)
  2. Pairwise preference annotation — Compare two model responses, pick
  preferred (both papers)
  3. Multi-turn conversational RLHF — Real-time multi-turn dialogue with
  comparison at each turn (Anthropic)
  4. Safety/red-teaming data collection — Adversarial prompts + safety
  classification (both papers)
  5. Response editing/rewriting — Annotators improve model outputs (Anthropic,
  late feature)
  6. Model evaluation (Elo/Arena) — Blind head-to-head model comparison
  (Anthropic Elo, LMSYS-style)
  7. Iterative campaign management — Multiple RLHF rounds with model improvement
   between batches (Meta V1-V5, Anthropic online)
  8. Annotator qualification & routing — Skill-based assignment with
  qualification tests (Meta 4-step)
