Platform module architecture for RLHF data collection

Researcher-owned
Task designer-owned
Annotator-facing
QA reviewer-owned
Automated + Researcher
Orchestration layer
Project hub
M1
The top-level container. A "project" = one client engagement (e.g., Anthropic's RLHF campaign). Contains iteration cycles, target metrics, data volume goals, and model version history. This is where the researcher defines "what are we trying to achieve this week" and tracks progress across iterations.
Addresses pain: 1.2 (strategy feedback), 1.3 (cross-iteration memory)
Configuration layer
Task studio
M2
Where task designers build annotation tasks: define guidelines, rubrics, evaluation dimensions, preference scales, risk categories, and attack vectors. Version-controlled — every guideline change is tracked and can be A/B tested.
Addresses pain: 1.1 (translation gap), 2.1 (versioning), 2.2 (rubric ambiguity), 2.3 (A/B testing)
Workforce hub
M3
Annotator lifecycle management: recruitment pipelines, multi-stage vetting (Meta's 4-test process), skill tiering, task routing, compensation tracking, communication channels, and demographics. Maps annotators to task types by capability.
Addresses pain: 4.2 (fatigue detection), 4.6 (feedback loops to annotators)
↓ Task configs + annotator assignments flow down ↓
Generation layer
Model gateway
M4
Connects to client model endpoints. Manages model variant deployment, temperature/sampling configs, response pair generation, and latency optimization. Tracks which model version generated each response for full provenance.
Addresses pain: 3.1 (infra), 4.4 (latency)
Prompt engine
M5
Manages prompt sourcing and diversity: human-written prompts, model-generated synthetic prompts, adversarial seed prompts. Tracks topic/category coverage, detects distribution gaps, and ensures diversity targets are met per batch.
Addresses pain: 3.2 (diversity), 3.3 (quality gate)
↓ Prompts + model responses flow into annotation ↓
Core execution layer
Annotation workbench
M6
The annotator-facing interface. Supports multiple task modes: SFT demonstration writing, pairwise preference comparison with strength ratings, multi-turn conversational annotation, red-team adversarial probing, and safety labeling. Includes conversation branching, response editing (as Anthropic allowed for Upworkers), and real-time quality signals. This is the heart of the platform.
Addresses pain: 4.1 (cognitive load), 4.3 (signal degradation), 4.5 (multi-turn), 4.6 (real-time feedback)
↓ Raw annotations flow into quality control ↓
Quality & processing layer
Quality control center
M7
QA review workflows (approve/edit/reject), inter-annotator agreement monitoring, automated quality scoring using client reward models, drift detection over time, and calibration checks. Feeds quality signals back to Workforce Hub and Task Studio.
Addresses pain: 5.1 (QA bottleneck), 5.2 (rejection cost), 2.2 (ambiguity detection)
Data engine
M8
Data processing pipeline: formatting for training, train/test splitting, cross-batch mixing with configurable ratios, deduplication, distribution analysis, coverage reporting. Exports to client training infrastructure in their required format.
Addresses pain: 5.3 (ad-hoc mixing), 5.4 (distribution analysis)
↓ Processed data exports to client + results feed back ↑
Intelligence layer
Analytics & insights
M9
Cross-iteration analytics: reward model accuracy trends, data quality over time, annotator performance, coverage gaps, false refusal rates, cost-per-annotation, throughput metrics. Generates actionable recommendations for next iteration strategy. Closes the loop back to Project Hub.
Addresses pain: 6.1 (RM accuracy visibility), 6.2 (slow feedback), 6.3 (eval cost)
